Getting Started

Get up and running with NextAssist in minutes.

NextAssist is an AI chatbot assistant integrated into your Frappe site. Access it at /nextassist from your browser.

Requirement: You must have the System Manager role to access NextAssist.

The main navigation sidebar includes:

  • Chat — Start and continue AI conversations
  • Sessions — View and manage chat history
  • Schedulers — Set up recurring automated tasks
  • Providers — Configure AI provider integrations
  • Settings — Application-wide settings

Setting Up AI Providers

Configure AI providers to power your conversations.

Before using the chat, configure at least one AI provider.

Adding a New Provider

  1. Navigate to Providers from the sidebar
  2. Click New Provider
  3. Fill in the required fields:
FieldDescription
Provider NameA unique name (e.g., "my-openai")
Provider TypeOpenAI, Anthropic, or Google
API KeyYour provider's API key (stored encrypted)
Default ModelSelect from the dropdown for your provider type
Max TokensMaximum tokens per response (default: 4096)
TemperatureResponse randomness, 0–2 (default: 0.7)
Max Context MessagesPrevious messages included as context (default: 20)
API Base URLOverride the default endpoint (optional)
Organization IDRequired for some OpenAI accounts (optional)

Toggle Enabled to activate, and Default Provider to make it the default for new chats. Click Create to save.

Managing Providers

  • Edit — Click a provider from the list to update settings
  • View/Copy API Key — Use the eye icon to reveal the key, copy icon to copy
  • Delete — Click Delete on the provider detail page

Application Settings

Configure app-wide options for NextAssist.

Navigate to Settings to configure app-wide options:

SettingDescription
Default ProviderWhich AI provider to use for new chats
Enable Tool CallingAllow the AI to query data, create records, etc.
Enable File UploadsAllow file attachments in chat messages

Chat

Start AI-powered conversations with your business data.

Starting a Conversation

  1. Navigate to Chat from the sidebar
  2. Click New Chat
  3. Type your message and press Enter to send

Use Shift+Enter for a new line without sending.

Switching Models

Click the model badge above the input to open the model switcher. Models are grouped by provider type. You can change models mid-conversation.

AI Responses

Responses stream in real-time and can include:

  • Text — Formatted markdown (headings, bold, links, code blocks)
  • Tables — Scrollable data tables with formatted columns
  • Charts — Bar, line, pie, and percentage charts
  • Lists & Bullets — Structured data summaries
  • Files — Images as thumbnails, other files as download links
  • Code — Toggle visibility with the "Show code" button

File Uploads

When enabled in Settings, attach files (PDFs, images, text) to your messages. The AI reads and analyzes the file content.

Token Limits

80% usage — Warning banner appears. 100% — Session marked "Limit Reached", start a new chat.

Sessions

View and manage your chat history.

Navigate to Sessions to view all chat sessions.

Session List

Shows title, user, status (Active/Limit Reached/Archived), provider, model, token count, and last message time. Use search and status filters to find sessions.

View Chat (Preview)

Click View Chat to open a full-screen preview that renders the conversation exactly as it appeared — including charts, tables, formatted text, and errors. The preview has three columns:

  • Left — Session details and statistics
  • Center — Chat messages with full rendering
  • Right — Message timeline with timestamps

Actions

  • View Chat — Preview the conversation in a modal
  • Open in Chat — Continue the conversation
  • Delete — Remove the session and all messages

Schedulers

Automate recurring tasks with cron-based scheduling.

Create recurring automated tasks that run on a cron schedule.

Creating a Scheduler

  1. Navigate to Schedulers and click New Scheduler
  2. Set the title and cron expression (e.g., 0 9 * * * for daily at 9 AM)
  3. Configure the query: DocType, filters, fields, and optional condition
  4. Choose an action type and configure it

Action Types

TypeDescription
EmailSend an email with matched data
NotificationCreate a Frappe notification
WebhookPOST data to an external URL
Custom CodeExecute custom Python code

Monitoring

Each scheduler shows run history with status (success/error/skipped), duration, matched and actioned counts. Error details are preserved for debugging.

Provider Usage

Monitor AI provider usage and performance metrics.

Each provider's detail page shows a Usage Stats sidebar:

  • Tokens Burned — Total tokens used across all sessions
  • Active Sessions — Currently active sessions
  • Total Sessions — All sessions that used this provider
  • Last Used — When the provider was last used

The Recent Sessions list shows the latest sessions with status indicators, user info, and token counts. Click a session title to open it in chat.

Tips

Best practices for getting the most out of NextAssist.

  • Dark Mode — Toggle Light/Dark/System theme from the sidebar
  • Keyboard — Enter to send, Shift+Enter for new line
  • Context Window — Start a new chat when you hit the token limit for best results
  • Multiple Providers — Configure several providers and switch between them via Settings or the model switcher
